# 获取 Trace 的方法

目标：用 Spike 运行 RISC-V 程序，得到指令级 Trace，再转为二进制供 Rust 模拟器消费。

## Step 1：编写测试汇编

在 `test_binary/test_addi.S` 中写一个极简汇编，只包含几条 `addi`：

```asm
.global _start
_start:
    li      a0, 0
    addi    a0, a0, 10
    addi    a1, a0, 20
    addi    a2, a1, 30

    # 写 tohost = 1 通知 Spike 正常退出
    la      t0, tohost
    li      t1, 1
    sw      t1, 0(t0)

.end:
    j       .end
```

> **注意**：裸机程序必须通过向 `tohost` 地址写 1 让 Spike 正常退出（exit code 0），否则 Spike 会卡在死循环无限刷日志。`tohost` 符号由 linker script 定义。

## Step 2：交叉编译

```bash
riscv64-unknown-linux-gnu-gcc -nostdlib -nostartfiles -static -no-pie -T baremetal.ld test_addi.S -o test_addi
```

关键参数：
- `-nostdlib -nostartfiles`：不链接 C 运行时，纯裸机
- `-static -no-pie`：**必须**，否则 Linux 目标工具链会插入动态装载器段（地址 `0x7ffff000`），Spike 无法加载
- `-T baremetal.ld`：自定义 linker script，将 `tohost`/`fromhost` 符号放在 `0x80001000`，让 Spike 能正确退出

`baremetal.ld` 内容：
```ld
OUTPUT_ARCH(riscv)
ENTRY(_start)

SECTIONS
{
    . = 0x80000000;

    .text : {
        *(.text.entry)
        *(.text .text.*)
    }

    .rodata : { *(.rodata .rodata.*) }
    .data :   { *(.data .data.*) }

    .bss : {
        __bss_start = .;
        *(.bss .bss.*)
        *(COMMON)
        __bss_end = .;
    }

    . = ALIGN(0x1000);
    tohost   = .;
    fromhost = . + 0x40;
}
```

## Step 3：用 Spike 跑出 Trace 日志

```bash
spike -l --log-commits test_addi 2> spike.log
```

关键参数：
- **`-l`**：输出每条指令的反汇编文本（如 `addi a1, a0, 20`）
- **`--log-commits`**：输出每条指令提交后的寄存器变化（如 `x11 0x1e`）
- **两者必须同时开启**：单独 `--log-commits` 不含反汇编；单独 `-l` 不含寄存器值
- **输出到 stderr**：必须用 `2>` 重定向，不是 `>`
- **不需要 `pk`**：裸机 ELF 直接 `spike -l --log-commits test_addi` 即可

实际输出（已验证，exit code = 0）：
```text
core   0: 0x0000000080000000 (0x00004501) c.li    a0, 0
core   0: 3 0x0000000080000000 (0x4501) x10 0x0000000000000000
core   0: 0x0000000080000002 (0x00000529) c.addi  a0, 10
core   0: 3 0x0000000080000002 (0x0529) x10 0x000000000000000a
core   0: 0x0000000080000004 (0x01450593) addi    a1, a0, 20
core   0: 3 0x0000000080000004 (0x01450593) x11 0x000000000000001e
core   0: 0x0000000080000008 (0x01e58613) addi    a2, a1, 30
core   0: 3 0x0000000080000008 (0x01e58613) x12 0x000000000000003c
```

每条指令产生两行：第一行是 **取指 + 反汇编**，第二行（带 `3` 标记）是 **提交后的寄存器写回**。

## 为什么不能直接用文本日志

1. **体积爆炸**：十亿条指令 → 上百 GB 文本
2. **解析极慢**：Rust 端用正则逐行解析，90% 时间花在字符串处理上
3. **信息未结构化**：给你的是 `addi`，需要自己映射为 `OpClass::IntegerAlu`

因此必须离线预处理为二进制，Rust 端零解析开销直接 `mmap` 使用。
