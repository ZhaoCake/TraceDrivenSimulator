# 从单周期到多周期

在这样的一个基于轨迹进行驱动的仿真中，从单周期改为多周期比实际的RTL要简单得多。改为流水线也同样不困难。这里我们先改为多周期。

改为五阶段（取指 IF、译码 ID、执行 EX、访存 MEM、写回 WB）多周期之后，因为每条指令需要花费 5 个周期顺序执行，我们期望的IPC会降到 0.2。

在我们的 Simulator 当中，我们将每条指令的推进过程拆分为了五大阶段子函数直接模拟其处理过程（`stage_fetch`, `stage_decode`, `stage_execute`, `stage_memory`, `stage_writeback`）。在现有的多周期模型中，一条指令按序通过这五个子函数，每个函数内仅让周期递增 `self.cycle += 1`，以保证对于指令来说“每推进一个阶段才能逐渐获取和确定结果”。当前执行 `cargo run` 后，输出 IPC 稳定为 `0.200`。

这种分阶段的时序推进模型能够为后续我们将这五个函数拆开异步重叠调用、研究流水线冒险提供坚实的基础接口。
