## TDD 重构与代码结构拆分

现在的项目结构已经被划分为了：
- `src/lib.rs`：导出 `trace` 和 `simulator` 模块
- `src/trace.rs`：存放与文件交互直接映射的结构（`OpType`, `TraceRecordRaw`, `TraceRecord`）。提取了完全由结构体负责的抽象逻辑。
- `src/simulator.rs`：存放模拟器的实现 (`Simulator`)。并已经 **内置了基于 `[#cfg(test)]` 的多条 TraceRecord 假指令推演测试用例**。
- `src/main.rs`：只专注于输入参数的读写和系统流串联调用（将 `File` 内存通过 `unsafe` 和 `std::ptr::read_unaligned` 转码后扔给 `Simulator`）。

### Simulator 中的内置 TDD 测试用例

我们利用伪造的简单 `vec![TraceRecord]`，无需任何 `.trace` 文件便可对 `step` 操作和核心 `op_stats` 状态预推进进行自动化测试。

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_simulator_step_and_stats() {
        let mut sim = Simulator::new();

        // 构造虚构的 TraceRecord，测试模拟器内部运行逻辑
        let trace_sequence = vec![
            TraceRecord { rs1: 0, rd: 1, op_type: OpType::IntAlu, .. },
            TraceRecord { rs1: 1, rd: 2, op_type: OpType::IntAlu, .. },
            TraceRecord { rs1: 2, rd: 0, op_type: OpType::Store,  .. }
        ];

        for record in &trace_sequence {
            sim.step(record);
        }

        // 测试基础统计指标
        assert_eq!(sim.cycle, 3);
        assert_eq!(sim.inst_count, 3);
        assert_eq!(*sim.op_stats.get(&OpType::IntAlu).unwrap_or(&0), 2);
        assert_eq!(*sim.op_stats.get(&OpType::Store).unwrap_or(&0), 1);
    }
}
```

通过运行 `cargo test` 及 `cargo run`，不仅做到了底层和二进制输入流的解耦，为往其中塞各种复杂超标量队列、依赖流图提供了极佳的扩展性和安全性保障。
