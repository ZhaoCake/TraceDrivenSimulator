# 使用CPP重写该项目

为了加快开发节奏，现放弃使用rust，改用cpp进行开发。

重写该项目为cpp 17标准。需求使用gtest代替cargo test，并保留各模块逻辑，但完全转化为cpp所使用的方式。需要使用的环境使用flake.nix描述，并且删除原本rust环境的相关内容。文件后缀全部使用.cc，注释全部用中文。

重写成果回复写入到该文件中。 [重写成果](#重写成果)之前的内容不可更改。

### Rust → C++ 惯用法映射

| Rust 构造 | C++17 等价 | 示例 |
|-----------|-----------|------|
| `Option<T>` | `std::optional<T>` | 流水线锁存器字段 |
| `Vec<T>` | `std::vector<T>` | `load_trace()` 参数 |
| `VecDeque<T>` | `std::deque<T>` | Trace 输入队列 |
| `HashMap<K, V>` | `std::unordered_map<K, V>` | `op_stats` 指令分布 |
| `[u64; 32]` | `std::array<uint64_t, 32>` | 寄存器文件 |
| `#[repr(C, packed)]` | `__attribute__((packed))` | `TraceRecordRaw` 二进制布局 |
| `enum with u64 payload` | `struct { Kind enum; uint64_t value; }` | `ForwardSource` 标签联合体 |
| `match` | `switch` / `if-else` | ALU compute、HazardUnit |
| `.clone()` | 拷贝构造 (隐式) | `PipelineSnapshot::snapshot()` |
| `.take()` | `std::move` + `= std::nullopt` | 锁存器推进 (reverse commit) |
| `#[test]` | `TEST()` 宏 | gtest 测试用例 |
| `println!` | `std::cout` | `print_statistics()` |
| `&mut self` (修改统计) | 非 const 成员函数 | `forward_value()` |
| `impl From<u8>` | 自由函数 `op_type_from_u8()` | OpType 转换 |

### 关键实现细节

1. **打包结构体 (TraceRecordRaw)**: 使用 `__attribute__((packed))` 确保与 Rust `#[repr(C, packed)]` 相同的二进制布局，保证从 Trace 二进制文件中正确读取。

2. **ForwardSource 标签联合体**: Rust 中 `enum ForwardSource { RegFile(u64), ExMemAlu(u64), ... }` 所有变体携带相同类型 `u64`，在 C++ 中用 `struct ForwardSource { ForwardSourceKind kind; uint64_t value; }` 实现，配合 `operator==` 支持 gtest 的 `EXPECT_EQ`。

3. **两阶段流水线推进**: 严格保留 Rust 的两阶段设计 — 快照 (snapshot) → 组合逻辑 (HazardUnit::resolve) → 反向提交 (WB→MEM→EX→ID→IF)。反向顺序实现同周期 EX→ID 数据转发。

4. **std::optional 移动语义**: Rust 的 `Option::take()` 模式用 `std::move(opt.value())` + `opt = std::nullopt` 实现，确保流水线锁存器的正确推进。

5. **build/ 目录**: CMake 构建产物输出到 `build/` 目录，已加入 `.gitignore`。编译器使用 `g++`（通过 flake.nix 的 `stdenv` 提供），IDE 支持通过 `clang-tools` 包提供 `clangd`，配合 `CMAKE_EXPORT_COMPILE_COMMANDS=ON` 生成 `compile_commands.json`。

### 构建与运行

```bash
# 进入 Nix 开发环境
nix develop

# 配置与构建
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build

# 运行测试
ctest --test-dir build

# 运行模拟器
./build/trace_simulator <trace二进制文件>
```
